enum Keyword

Enumeration of language keywords.

This enumeration defines the reserved keywords in the language and their corresponding roles in syntax. Keywords are case-sensitive and cannot be used as identifiers.

Enumerator
KEYWORD_IF 

Represents the if keyword. Used for conditional branching to execute code based on a boolean condition.

KEYWORD_ELSE 

Represents the else keyword. Specifies the block of code to execute if the condition in the corresponding if statement is false.

KEYWORD_FN 

Represents the fn keyword. Used to define a function in the language.

KEYWORD_CONST 

Represents the const keyword. Declares a constant value that cannot be modified after initialization.

KEYWORD_I32 

Represents the i32 keyword. Specifies a 32-bit signed integer type.

KEYWORD_F64 

Represents the f64 keyword. Specifies a 64-bit floating-point type.

KEYWORD_U8 

Represents the u8 keyword. Specifies an 8-bit unsigned integer type.

KEYWORD_NULL 

Represents the null keyword. Denotes a null or uninitialized value.

KEYWORD_PUB 

Represents the pub keyword. Indicates public visibility for functions, variables, or constants, allowing access from outside their scope.

KEYWORD_RETURN 

Represents the return keyword. Used to exit a function and optionally provide a return value.

KEYWORD_VAR 

Represents the var keyword. Declares a variable that can be reassigned during its lifetime.

KEYWORD_VOID 

Represents the void keyword. Specifies that a function does not return a value.

KEYWORD_WHILE 

Represents the while keyword. Creates a loop that repeatedly executes code as long as a boolean condition evaluates to true.

enum State

Enumeration representing the states of a finite state machine (FSM) for lexical analysis.

This FSM is used to tokenize input by transitioning between states based on the input character. Each state corresponds to a specific type of token or behavior during tokenization.

Enumerator
sStart 

The initial state of the FSM. This state represents the starting point before processing any input character.

sIdentifierorKeyword 

States for handling identifiers and keywords.

Transition state for detecting identifiers or keywords; case-sensitive.

sKeyword 

Indicates the token is a reserved keyword (e.g., const, else, fn, if, etc.).

sIdentifier 

Indicates the token is an identifier (e.g., variable or function name).

sIntLiteral 

States for handling numeric literals.

Represents an integer literal (non-negative, decimal format).

sFloatLiteral 

Represents a floating-point literal, including support for decimal and exponential notation.

sExponent 

Handles the exponent part (e.g., "e" or "E") of floating-point numbers.

sStringLiteral 

States for handling string literals.

Represents a string literal enclosed in double quotes.

sEscapeSequence 

Handles escape sequences (e.g.,
, \t) within string literals.

sBackSlash 

States for handling operators and punctuation.

Handles the backslash character ('\').

sDivision 

Represents the division operator ('/').

sAssign 

Represents the assignment operator ('=').

sEqual 

Represents the equality comparison operator ('==').

sNotEqual 

Represents the inequality operator ('!=').

sLessEqual 

Represents the less than or equal to operator ('<=').

sLessThan 

Represents the less than operator ('<').

sGreaterEqual 

Represents the greater than or equal to operator ('>=').

sGreaterThan 

Represents the greater than operator ('>').

sExponentSign 

Represents the sign (e.g., '+' or '-') within an exponent part of a floating-point literal.

sSingleQuote 

States for handling quotation marks and character literals.

Represents a single quote character (') used in character literals.

sLeftSingleQuote 

Represents a left single quotation mark (‘), often used in specialized typography.

sExclamation 

Represents the exclamation mark ('!'), used in logical negation or other operators.

sComment 

State for handling comments.

Represents a line comment starting with "//".

sImport 

Represents the '@' symbol for import statements.

sError 

States indicating errors or the end of input.

Indicates an error state due to invalid input or characters.

sEOL 

End-of-line state for newline characters.

sEnd 

End-of-file state or input completion.

enum Token_type

Enumeration of possible token types.

This enumeration defines the various types of tokens that can be identified during lexical analysis. Each token type represents a specific category of syntactic element in the language.

Enumerator
TOKEN_EOF 

End-of-file marker. Indicates that the end of the input file has been reached.

TOKEN_EOL 

End-of-line marker. Represents a newline character in the source code.

TOKEN_IDENTIFIER 

Identifier token. Represents variable names, function names, or other user-defined identifiers.

TOKEN_KEYWORD 

Keyword token. Represents a reserved keyword in the language (e.g., if, else, return).

TOKEN_NEWLINE 

Newline character token. Specifically used for handling newline characters (\n) in the input.

TOKEN_TAB 

Tab character token. Represents a tab character (\t) in the input.

TOKEN_INT_LITERAL 

Integer literal token. Represents whole numbers (e.g., 42, -7).

TOKEN_FLOAT_LITERAL 

Floating-point literal token. Represents decimal or exponential numbers (e.g., 3.14, 2.5e-10).

TOKEN_STRING_LITERAL 

String literal token. Represents a sequence of characters enclosed in double quotes.

TOKEN_BOOLEAN 

Boolean literal token. Represents a boolean value (true or false).

TOKEN_CHAR_LITERAL 

Character literal token. Represents a single character enclosed in single quotes (e.g., ‘'a’`).

TOKEN_EQUAL 

Equality comparison operator. Represents the equality operator (==).

TOKEN_NOT_EQUAL 

Inequality comparison operator. Represents the not-equal-to operator (!=).

TOKEN_LESS_EQUAL 

Less-than-or-equal-to operator. Represents the operator (<=).

TOKEN_LESS_THAN 

Less-than operator. Represents the operator (<).

TOKEN_GREATER_EQUAL 

Greater-than-or-equal-to operator. Represents the operator (>=).

TOKEN_GREATER_THAN 

Greater-than operator. Represents the operator (>).

TOKEN_ASSIGNMENT 

Assignment operator. Represents the assignment operator (=).

TOKEN_ADDITION 

Addition operator. Represents the addition operator (+).

TOKEN_SUBTRACTION 

Subtraction operator. Represents the subtraction operator (-).

TOKEN_MULTIPLY 

Multiplication operator. Represents the multiplication operator (*).

TOKEN_DIVISION 

Division operator. Represents the division operator (/) with a result as a floating-point number.

TOKEN_BACKSLASH 

Backslash character token. Represents the backslash character (\).

TOKEN_PIPE 

Logical OR operator. Represents the logical OR operator (||).

TOKEN_EXCLAMATION 

Exclamation mark token. Represents the exclamation mark (!), typically used in logical negation.

TOKEN_LPAREN 

Left parenthesis token. Represents the opening parenthesis (().

TOKEN_RPAREN 

Right parenthesis token. Represents the closing parenthesis ()).

TOKEN_LEFT_BRACKET 

Left square bracket token. Represents the opening square bracket ([).

TOKEN_RIGHT_BRACKET 

Right square bracket token. Represents the closing square bracket (]).

TOKEN_CURLYL_BRACKET 

Left curly bracket token. Represents the opening curly bracket ({).

TOKEN_CURLYR_BRACKET 

Right curly bracket token. Represents the closing curly bracket (}).

TOKEN_COMMA 

Comma token. Represents the comma (,) used for separating elements.

TOKEN_SEMICOLON 

Semicolon token. Represents the semicolon (;) used for statement termination.

TOKEN_COMMENT 

Comment token. Represents comments in the code (e.g., // This is a comment).

TOKEN_IMPORT 

Import token. Represents the @import statement.

TOKEN_DISCARD 

Discard token. Represents the underscore (_), used for discarding values.

TOKEN_COLON 

Colon token. Represents the colon (:).

TOKEN_DOT 

Dot token. Represents the dot (.), often used for member access or namespaces.

TOKEN_QUESTION_MARK 

Question mark token. Represents the question mark (?), often used in conditional expressions.

TOKEN_UNDEFINED 

Undefined token. Represents an unrecognized or invalid token.

TOKEN_STRINGINIT_ERROR 

String initialization error token. Represents an error encountered during string initialization.
